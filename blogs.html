Active Directory: Create New User Programatically

New-ADUser in a specific OU


One of the most repetitive tasks System Administrators perform on a daily basis is the creation of user accounts. There are several approaches the technician can create a new account in Active Directory. Some prefer doing it through the User Interface(GUI), quick and simple. All one needs to do is to fill out the fields or attributes following the standards of the business.

But for some of us, who don't want to spend a few of our precious minutes; we like to make it quick and simple.

To my fellow engineers out there, here's a script I have derived to automate the task:


I will start by assigning static values to my variables, in case you're wondering why there are only 3 variables; We will get to the explanation later in the details below.

        
    $gname = 'John'
    $lname = 'DOE'
    $acctpwd = "whateverP@ssw0rd"

On the next part, I will use a reference account which is an existing user in the AD and store it in a variable called "$ref_User". I use this technique to retrieve the user account details I will copy to the new user account. The only thing to note here is that the new account will be in the same department or role of the reference account. 

        
    $ref_User = jane.dove

I will also retrieve the domain name of the AD Forest. Then store "that" value in a variable call "$upnsfx", which will be used later on the script as the UPN Suffix.

        
    $upnsfx = (Get-ADForest).UPNSuffixes

I will also need the OU location where the new account will be added. The 1st line will retrieve all the attributes of the reference account. The 2nd line will select only the DistinguishedName attribute of the same reference account. The 3rd line will isolate the string value of the Organizational Unit (OU) of that same reference account.

        
    $rfrnc_user = Get-ADUser $ref_User -Properties *
    $dsn = ($rfrnc_user).DistinguishedName
    $targetOU = $dsn.Substring($dsn.IndexOf('OU=',[System.StringComparison]::CurrentCultureIgnoreCase))

And lastly I will assign all values that was stored in those variables into the new user account attributes.

        
    $splat = @{
        Path = $targetOU
        Name = "$($gname) $($lname)"
        DisplayName = "$($gname) $($lname)"
        GivenName = $gname
        SurName = $lname
        UserPrincipalName = "$(($gname).ToLowerCase()).$(($lname).ToLowerCase)@$($(upnsfx))"
        Description = 'System Administrator' #depends on what your organization use this attribute
        AccountPassword = (ConvertTo-SecureString -AsPlainText $acctpwd -force)	
    }
        
    New-ADUser @splat -Verbose

Combine all those components to make a whole script.


    $gname = 'John'
    $lname = 'DOE'
    $acctpwd = "whateverP@ssw0rd"
    $ref_User = jane.dove
    $upnsfx = (Get-ADForest).UPNSuffixes
    $rfrnc_user = Get-ADUser $ref_User -Properties *
    $dsn = ($rfrnc_user).DistinguishedName
    $targetOU = $dsn.Substring($dsn.IndexOf('OU=',[System.StringComparison]::CurrentCultureIgnoreCase))
    $splat = @{
        Path = $targetOU
        Name = "$($gname) $($lname)"
        DisplayName = "$($gname) $($lname)"
        GivenName = $gname
        SurName = $lname
        UserPrincipalName = "$(($gname).ToLowerCase()).$(($lname).ToLowerCase)@$($(upnsfx))"
        Description = 'System Administrator' #depends on what your organization use this attribute
        AccountPassword = (ConvertTo-SecureString -AsPlainText $acctpwd -force)	
    }

    New-ADUser @splat -Verbose

When the script is executed as-is, the result would return these values after you query using "Get-ADUser -Identity john.doe".


    Name -> John DOE
    DisplayName -> John DOE
    GivenName -> John
    SurName -> DOE
    UserPrincipalName -> john.doe@domain.xyz
    DistinguishedName -> CN=John DOE,OU=IT,OU=User,DC=domain-xyz,DC=local
