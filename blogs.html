Create New User in a Specific OU in Active Directory using PowerShell


One of the most repetitive tasks System Administrators perform on a daily basis is the creation of user accounts. There are several approaches the tehcnicia can create a new account in Active Directory. Some prefer doing it through the User Interface(GUI), quick and simple. All one needs to do is to fill out the fields or attributes following the standards of the business.

But for some of us, who don't want to spend a few of our precious minutes; we like to make it simple.

To my fellow engineers out there, here's a script I have derived to automate the task:


We start by assigning values to our variables which will store the most basic attributes for the new account

        
            $gname = 'John'
            $lname = 'DOE'
            $acctpwd = "whateverP@ssw0rd"

On the next part, we will create another variable and use a reference account which our new account will become part of that team/department/role.

        
            $ref_User = jane.dove

Create another variable and retrieve the UPN Suffix of a single domain forest which will be used later on the script.

        
            $upnsfx = (Get-ADForest).UPNSuffixes

To isolate the value of Organizational Unit (OU) of the reference account, we need to remove its Canonical Name

        
            $rfrnc_user = Get-ADUser $ref_User -Properties *
            $dsn = ($rfrnc_user).DistinguishedName
            $targetOU = $dsn.Substring($dsn.IndexOf('OU=',[System.StringComparison]::CurrentCultureIgnoreCase))

And lastly we assign all values retrieved into the account attributes

        
            $splat = @{
            Path = $targetOU
            Name = "$($gname) $($lname)"
            DisplayName = "$($gname) $($lname)"
            GivenName = $gname
            SurName = $lname
            UserPrincipalName = "$(($gname).ToLowerCase()).$(($lname).ToLowerCase)@$($(upnsfx))"
            Description = 'System Administrator' #depends on what your organization use this attribute
            AccountPassword = (ConvertTo-SecureString -AsPlainText $acctpwd -force)	
            }
    
            New-ADUser @splat -Verbose

Combine all those components to make a whole script


            $gname = 'John'
            $lname = 'DOE'
            $acctpwd = "whateverP@ssw0rd"
            $ref_User = jane.dove
            $upnsfx = (Get-ADForest).UPNSuffixes
            $rfrnc_user = Get-ADUser $ref_User -Properties *
            $dsn = ($rfrnc_user).DistinguishedName
            $targetOU = $dsn.Substring($dsn.IndexOf('OU=',[System.StringComparison]::CurrentCultureIgnoreCase))
            $splat = @{
                Path = $targetOU
                Name = "$($gname) $($lname)"
                DisplayName = "$($gname) $($lname)"
                GivenName = $gname
                SurName = $lname
                UserPrincipalName = "$(($gname).ToLowerCase()).$(($lname).ToLowerCase)@$($(upnsfx))"
                Description = 'System Administrator' #depends on what your organization use this attribute
                AccountPassword = (ConvertTo-SecureString -AsPlainText $acctpwd -force)	
            }
    
            New-ADUser @splat -Verbose

When the script is executed as-is, the result would return these values after you query using Get-ADUser -Identity john.doe


            Name -> John DOE
            DisplayName -> John DOE
            GivenName -> John
            SurName -> DOE
            UserPrincipalName -> john.doe@domain.xyz
            DistinguishedName -> CN=John DOE,OU=IT,OU=User,DC=domain-xyz,DC=local
